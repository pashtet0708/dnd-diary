# Как обновлять сайт-дневник

Сайт лежит на **GitHub Pages**. Любая правка в репозитории → сайт сам обновляется
через ~1 минуту. После обновления жми **Ctrl+F5** (сбросить кэш браузера).

Где что лежит:
- **`story.js`** — сам сюжет, истории по дням. ← правят чаще всего
- **`data.js`** — название сайта, отряд, журнал заданий
- `app.js`, `styles.css`, `index.html` — движок и оформление, трогать не надо
- `images/` — картинки (лёгкие `.webp`)

---

## A. Добавить / изменить главу сюжета — `story.js`

Проще всего прямо на GitHub: открыл `story.js` → ✏️ Edit → правишь → Commit.

Добавить новый день:
1. Скопируй один блок `{ ... }` (от `{` до `},`) и вставь **в конец** массива,
   перед закрывающей `];`.
2. Поменяй `day`, `date`, `title`, `location`, `text`.
   Новый абзац — `\n\n` между кусками текста.
3. Картинки — раздел C.

> ⚠️ Каждая запись заканчивается запятой `,`. Кавычки `"` — парами.
> Если сайт «сломался» — ищи лишнюю/пропущенную кавычку или запятую.

---

## B. Поменять отряд или задания — `data.js`

**Отряд** — блок `const PARTY = [ ... ]`. Герой:
```js
{ name: "Имя", race: "Раса", cls: "Класс", color: "#7a2f2f" },
```
`cls` — по классу сам подбирается значок. У новичков можно добавить
`joined: "присоединился с 3-го дня"`.

**Журнал заданий** — блок `const QUESTS = [ ... ]`. Задание:
```js
{ title: "Название", hint: "Описание / откуда узнали" },
```
Выполнено — допиши `done: true,` — карточка станет приглушённой с пометкой «раскрыто».

---

## C. Добавить картинки

1. Положи картинки (PNG/JPG) в папку `images/` локально.
2. **Сожми для веба** — в терминале из папки сайта:
   ```
   python compress_images.py
   ```
   Сделает лёгкие `.webp`, оригиналы спрячет в `images/_original/`.
   (Один раз: `pip install pillow`, если ругается.)
3. Пропиши в нужной главе `story.js`, расширение **`.webp`**:
   ```js
   images: [
     { src: "images/моя.webp",     cap: "Подпись", side: "right" },
     { src: "images/широкая.webp", cap: "Баннер",  full: true },
   ],
   ```
   - `side: "left"`/`"right"` — текст обтекает картинку.
   - `full: true` — полоса во всю ширину, **только для широких** картинок.
   - Узкие/мелкие — всегда `side` (иначе размыто).

> Картинки через веб GitHub грузить можно, но они **не сожмутся**. Лучше локально.

---

## D. Опубликовать

**Правил на GitHub в браузере?** Ничего больше не нужно — Commit, и через ~1 мин живёт.

**Правил локально (картинки/файлы)?** Из папки сайта:
```powershell
cd D:\Claude\dnd-diary
git add -A
git commit -m "что поменял"
git push
```
Через ~1 мин обновится. Прогресс — вкладка **Actions** в репозитории (зелёная галочка).

Адрес сайта: `https://pashtet0708.github.io/dnd-diary/`

---

## Если что-то пошло не так

- **Сайт «сломался» после правки `story.js`/`data.js`** — почти всегда лишняя/
  пропущенная кавычка `"` или запятая `,`. Открой файл на GitHub — он подсветит.
- **Старое показывается** — кэш. Ctrl+F5.
- **Проверить локально перед заливкой** — из папки сайта `python -m http.server 8000`,
  открой `http://localhost:8000`.
